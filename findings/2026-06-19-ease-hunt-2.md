# Ease-hunt #2 — fresh-angle wins (selector-resilience, read-economy, waits, grids, errors) — 2026-06-19

Second ease-hunt on angles the first didn't cover. 5 candidates, zero overlap with hunt #1. ALL FIVE shipped
(verified live; the read_table one was initially deferred for "no live grid" — I found one). Owner directive:
"you're the end user — if it helps, add it." It all helps; I verified each before shipping.

## Shipped (all live-verified, pushed)

1. **perf(mcp): cap the rendered node NAME like value@40** — the node name was the ONE uncapped field in
   renderSnapshot/renderDiff (value@40, inspect-body@2000, read@4000 all capped). A 1162-char VS Code editor
   line rendered in FULL on every per-action snapshot. capLabel(120). Live-measured across the 12-window
   desktop: ~9% off the snapshot (9448 bytes), paid every action. Matching unaffected (selectors read live
   element.name). Unit test test/render-name-cap.test.ts.
2. **feat(mcp): name the current control in the stale-generation rejection** — resolveRef held the live
   Full-cached Element at the moment it rejected a stale ref, then discarded its identity. Now it names what
   the id denotes NOW (named() = zero round-trips) and hands back ${id}#${refGen}: one-call confirm-by-name
   self-heal vs re-reading the ~2k-token snapshot. Keeps "earlier snapshot generation" (epoch-refs compat).
   Verified live (taskbar + screenshot_marked).
3. **feat(mcp): wait_for can target a control by ref** — wait_for only took a selector and threw "empty
   selector — …or target by ref" (a false promise); findFirstMatch couldn't wait on a non-unique control. Added
   a ref branch (Element.waitForOwnState, reuses stateMatches) so a control with no unique selector can be
   awaited by the exact ref find_and_act acts on. gone stays selector-only. New test wait-for-ref.
4. **feat(mcp): attach-by-title no-match steers to list_windows + lists candidates** — the one attach error
   that dead-ended with the library's bare "no window found for {…}" (echoes the agent's JSON) while every
   sibling steers + lists candidate windows with hWnds. Now it lists them (keeping the all.length===0
   EnumWindows-miss fallback). Titles are the most-guessed attach key. New test attach-title-no-match.
5. **feat(mcp): read_table {startRow} pages a big grid** — the loop was top-anchored (row=0) with only maxRows;
   the footer steered "raise maxRows" → re-read from the top → O(pages^2). Additive startRow (default 0 =
   unchanged), reads rows [startRow, startRow+maxRows), footer names the next startRow. Verified live on
   Explorer's System32 grid (startRow:0→rows 1-4 "next startRow:4"; startRow:4→DIFFERENT rows 5-8). New test
   read-table-paging. (Note: a virtualized grid like Explorer only realizes its viewport rows, so paging is
   bounded to the realized set there; a non-virtualized >100-row grid pages fully. The mechanism + clamp are
   proven either way.)
